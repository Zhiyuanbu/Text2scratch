1. Syntax Basics

These rules prevent most errors and make scripts readable.

One line = one command

Each line becomes one Scratch action in generated blocks.

move 10

Indent nested commands

Loop or if content must be indented below its parent.

repeat 10 move 10 end

Close with end

Every open control block needs a closing end.

if var(score) > 5 say "Great" end

Use quotes for sentence text

If text has spaces, wrap it in quotes.

say "Hello team"

2. Build Order

Use this sequence to avoid memory overload and broken scripts.

  1. Create shared data first: make_var, make_list, make_broadcast.
  2. Write stage behavior in stage_code = ... end.
  3. Add each sprite with sprite = "Name".
  4. Write sprite scripts in name_code = ... end.
  5. Run conversion and fix any Problems before export.

3. @ Expression Syntax

@ means "insert a value returned by another block".

Correct usage

set_var score @var(best_score)

Incorrect usage

@var(score)

Readable condition

if var(score) > 5 say "Great" end

4. Stage Coding Rules

Some Scratch blocks are sprite-only and cannot run on stage.

  • Put stage scripts inside stage_code = only.
  • If a line is stage-incompatible, move it into a sprite code section.
  • Use broadcasts to coordinate stage and sprites cleanly.

5. Multi-Sprite Template

Use this as your starter file for most projects.

make_var score 0 make_broadcast start_round stage_code = when_flag_clicked switch_backdrop_to backdrop1 broadcast start_round end sprite = "Cat" cat_code = when_broadcast_received start_round set_var score 0 repeat 5 move 10 change_var score 1 end end sprite = "Ball" ball_code = when_broadcast_received start_round if var(score) > 3 say "Ready" else think "Wait" end end

6. Beginner Walkthrough (Copy, Test, Expand)

This section assumes you are brand new. Follow each step exactly, test, then move to the next step.

Step 1: Paste this exact starter script

Use this script first. Do not rename anything yet. Click Convert/Download only after this exact version works.

make_var score 0 make_broadcast game_start stage_code = when_flag_clicked set_var score 0 broadcast game_start end sprite = "Cat" cat_code = when_broadcast_received game_start forever wait 1 change_var score 1 end end

Step 2: Understand what each part does

  • make_var score 0: creates a variable named score starting at 0.
  • make_broadcast game_start: creates a message that stage and sprites can share.
  • stage_code = ... end: defines stage-only logic.
  • when_flag_clicked: runs when green flag starts the project.
  • set_var score 0: resets score every run (important).
  • broadcast game_start: tells sprites to start their logic.
  • sprite = "Cat": starts a sprite section for Cat.
  • cat_code = ... end: scripts that belong only to Cat.
  • forever ... end: loop runs continuously.

Step 3: Test checklist (must pass before step 4)

  1. Export and open the generated project in Scratch.
  2. Click green flag once.
  3. Confirm score increases by 1 every second.
  4. Click green flag again and confirm score restarts from 0.

If any item fails, do not continue. Fix this base first.

Step 4: Add win behavior at score 10

Replace only the Cat section with this:

cat_code = when_broadcast_received game_start forever wait 1 change_var score 1 if var(score) >= 10 say "You win!" stop_all end end end

Expected result: at score 10, Cat says "You win!" and the project stops.

Step 5: Add a second sprite safely

Append this new section at the bottom:

sprite = "Ball" ball_code = when_broadcast_received game_start forever move 6 if on_edge_bounce say "boing" end end end

Now both sprites start from the same broadcast. This is the basic multiplayer-script pattern.

Step 6: Common beginner errors and exact fixes

  • Error: "Unexpected end" or broken block tree. Fix: check every opened if, repeat, forever has a matching end.
  • Error: nothing happens when green flag starts. Fix: ensure stage has when_flag_clicked and it broadcasts the same name sprites listen to.
  • Error: score keeps old value on rerun. Fix: put set_var score 0 in startup logic.
  • Error: command works in sprite but not stage. Fix: move sprite-only command into a sprite section.
  • Error: parser rejects text with spaces. Fix: wrap text in quotes, for example say "Game Over".

Step 7: How to debug quickly

  1. Comment out one section at a time (or temporarily remove it).
  2. Run again and find the smallest section that causes failure.
  3. Add that section back line-by-line until error returns.
  4. Fix the exact line, then test end-to-end one more time.
