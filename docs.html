1. Quick Start Path

Use this sequence when starting a new game or animation project.

  1. Open converter and paste or write your script.
  2. Add setup first: variables, lists, and broadcasts.
  3. Place stage-only logic in stage_code = blocks.
  4. Create each sprite with sprite = "Name" and name_code =.
  5. Run export only after compiler output has no errors.

Starter Script

make_var score 0 make_broadcast game_start stage_code = when_flag_clicked set_var score 0 broadcast game_start end sprite = "Cat" cat_code = when_broadcast_received game_start forever wait 1 change_var score 1 end end

2. Syntax Basics

These rules prevent nearly all parser failures.

One line = one command

Keep each line focused. Do not chain commands on one line.

move 10

Indent nested bodies

Lines inside loops and conditions must be indented.

repeat 10 move 10 end

Always close with end

If you open if, repeat, or forever, close it.

if var(score) > 5 say "Ready" end

Quote text with spaces

Use quotes for labels and messages.

say "Game Over"

3. Project Structure

Use this template when multiple sprites and stage logic are involved.

make_var score 0 make_broadcast round_start stage_code = when_flag_clicked set_var score 0 broadcast round_start end sprite = "Player" player_code = when_broadcast_received round_start forever move 6 end end sprite = "Ball" ball_code = when_broadcast_received round_start if var(score) > 5 say "Go" end end

4. Expressions and @ Usage

@ marks a nested expression. Expressions are values, not standalone lines.

Correct

set_var score @var(best_score)

Incorrect

@var(score)

Readable Conditions

if var(score) >= 10 broadcast win end

5. Stage and Sprite Rules

Stage and sprite targets do not support the same command set.

  • Put only stage-compatible logic inside stage_code =.
  • Sprite motion/looks commands belong inside sprite code sections.
  • Use broadcasts to coordinate stage and sprite behaviors cleanly.
  • When in doubt, open Full Reference and filter by Works On.

6. Debugging and Error Recovery

Use this loop when conversion fails or warnings pile up.

  1. Fix every error from top to bottom before exporting again.
  2. If parser says missing end, check nearest opened block above it.
  3. If a command is unknown, verify spelling in Full Reference.
  4. If stage incompatibility appears, move that line to sprite code.
  5. Re-export only after errors are fully cleared.

Common Fix Patterns

  • Unknown command: correct syntax or comment line out with #.
  • Expression alone: embed reporter inside a real command input.
  • Invalid else: ensure it is inside an if body and appears once.
  • Stage incompatibility: move line into sprite section.

7. Sharing and Community Flow

Publish safely and keep ownership boundaries clear.

  1. Sign in from the Account page and save project to cloud.
  2. Create share link only when script is stable.
  3. Open community page to browse recent public projects.
  4. Shared projects load read-only unless you are the creator.
